## 1. プロジェクト概要

### サービス名
縁 enishi（えにし）

### ミッション
「なぜ」から始める Inside-Out パラダイムシフト支援。

「付き合う人・場所が大切」とは広く言われるが、なぜ本質的にそうなのかを説明できる場はほとんどない。縁 enishi は、エネルギーの層・相転移というメカニズムから「なぜ」を伝え、自分の内側から判断軸が育つ状態をつくる伴走者となる。

### 名前の意味
**縁（えにし）** ── 自分と何かが接触するその接点。  
どんな縁と接触し続けるかが、自分のエネルギーの相を決める。サービス名がそのままコンセプトを体現している。

### 現フェーズ
Docusaurus でモックを構築・検証中。本要件定義書をもとにオリジナル Web サイトとして再設計する。

## 2. コアフレームワーク

### 2-1. H₂O モデル（意識の相転移）

人間のエネルギー・意識には相（フェーズ）がある。H₂O が同じ物質でありながら氷・水・水蒸気になるように、人間の意識も層によって全く異なる性質を持つ。

| 相 | 意識の質 | 特徴 |
|----|---------|------|
| **氷** | 重く分断された意識 | 固定・防衛・「正しさ」への執着。変化に抵抗する |
| **水** | 柔軟・流動的な意識 | 動ける。ただし容れ物（環境・他者）の形に従いやすい |
| **水蒸気** | 共鳴する意識 | コントロールの枠を超え、広がり、共鳴でつながる |

相転移は意志力では起きない。**接触する層の変化**によって自然と起きる。

### 2-2. 接触の法則

> 自分が最も長く接触している層が、自分の相転移の条件になる。

これが「付き合う人・場所が大切」の本質的な理由。統計的観察ではなくメカニズムとして説明できる。場所・情報・言葉・空間・関係性、すべて含む。

### 2-3. 言語と層

言語は「どの層で受け取るか」によって意味が変わる。むしゃくしゃしているときに正論を言われてもだるいのは、言語（論理層）を氷の状態で受け取っているから。言葉の問題ではなく、接触している層の問題。

→ だから縁 enishi は「正論として伝える」のではなく「体感として届く言葉」を設計する。

### 2-4. 到達点の描写

外側がどうであれ「それも素敵だ」という視点が自然に生まれる状態。  
「あれはダメ、正さないと」という視点が消える──消そうとするのではなく、自然と消える。  
外を攻撃することが自分を攻撃することだと、体感レベルでわかる。

そのとき人は**気楽になる**。気が、楽になる。  
（日本語の「気楽」が示す通り、これはエネルギー層での解放）

## 4. ファネル設計

```
[ 認知 ]  SNS（X・Instagram等）/ note
    ↓ 「なぜ」という問いに共鳴した人が自発的に動く
[ 理解 ]  縁 enishi Web サイト
    ↓ メカニズムを理解し、信頼を持つ
[ 接点 ]  LINE 登録（個人窓口 / 経営者窓口）
    ↓ 双方向の関係が始まる
[ 転換 ]  個別セッション・伴走契約（有料）
```

**ポリシー：**
- 料金はサイト上に掲載しない。LINE でのみ案内する
- セッション案内は「この人なら」と判断した場合のみ
- バズより深い共鳴を優先。ターゲット以外が離れることを恐れない
- 占い・スピリチュアル・宗教・自己啓発を批判しない。VS 構造は一切作らない

## 6. デザイン方針

### トーン
- 知的・静謐・深み。自己啓発感・コーチング業界感を出さない
- 「答えを渡す」のではなく「問いの空間を作る」デザイン
- 体感に近い言葉。正論として届けない

### ビジュアル方向
- ダーク or ディープなカラーパレット（深海・宇宙・森）
- タイポグラフィ重視。余白を広く取る
- アニメーションは最小限。静けさを大切にする

### 避けること
- 自己啓発・コーチング業界の典型デザイン（明るい緑・ポジティブ系フォント）
- CTA の多用・圧力的な訴求
- 実績・資格・メディア掲載の羅列
- VS 構造・批判・比較

## 8. コンテンツ方針

### 書き方の原則
- 「教える」ではなく「問いを共有する」文体
- 1ページ = 1つの問い・1つの視点
- 「なぜそうなのか」のメカニズムまで届かせる
- 体感として読める言葉を選ぶ（氷層ではなく水・水蒸気層に届ける）
- VS 構造・批判・比較を使わない

### 更新フロー
```
SNS・note で発信した内容
  ↓ 反応が良かったもの・深めたいもの
サイトのページとして再編集・掲載
```
