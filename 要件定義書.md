# 要件定義書 — 縁 enishi Web サイト

**バージョン：** 0.1（初稿）
**作成日：** 2026-04-27
**ステータス：** モック検証中 → 本サイト設計へ

---

## 1. プロジェクト概要

### サービス名
縁 enishi（えにし）

### ミッション
「Being を起点に Inside-Out で生きる」ことを選択しようとしている人の伴走者となる。哲学・量子力学・AI活用・エネルギーという独自の切り口で、パラダイムシフトを支援する。

### 現フェーズ
Docusaurus でコンテンツ構造・ファネル設計を検証するモックを構築済み。
本要件定義書をもとに、オリジナル Web サイトとして再設計する。

---

## 2. ターゲットユーザー

### Persona A ─ 個人（自己探求層）

| 項目 | 内容 |
|------|------|
| 年代 | 30〜50代 |
| 状態 | 外側は整っているが内側が満たされない。自己啓発・コーチングを試みたが根本が変わらない感覚がある |
| 関心 | 哲学・量子力学・占い・AI・宇宙観 |
| 求めるもの | 「これだ」という腑落ち感。自分の在り方を探求できる場と伴走者 |

### Persona B ─ 経営者・法人

| 項目 | 内容 |
|------|------|
| 年代 | 40〜55代、経営者・役員・起業家 |
| 状態 | 物質的成功を達成。しかし虚無感や「次の問い」を抱えている |
| 関心 | 世界の哲学的転換・経営の在り方・組織変革 |
| 求めるもの | 誰にも言えない問いを扱える思考パートナー。月次顧問的関与 |
| 価格感 | 月 10〜30 万円 |

---

## 3. ファネル設計

```
[ 認知 ]  SNS（X・Instagram等）/ note
    ↓ 世界観に共鳴した人が自発的に動く
[ 理解 ]  縁 enishi Web サイト
    ↓ コンセプト・フレームワークを読み、信頼を持つ
[ 接点 ]  LINE 登録（個人窓口 / 経営者窓口）
    ↓ 双方向の関係が始まる
[ 転換 ]  個別セッション・伴走契約（有料）
```

**ポリシー：**
- 料金はサイト上に掲載しない。LINE でのみ案内する
- セッションへの案内は「この人なら」と判断した場合のみ行う
- バズより深い共鳴を優先する。ターゲット以外が離れることを恐れない

## 4. サイト構造

### 4-1. ページ構成

```
/                      トップページ
/concept               コンセプト概要
  /inside-out          Inside-Out とは
  /being               Being とは
  /paradigm-shift      パラダイムシフトの地図
  /ai-exploration      AI × 自己探求
/framework             フレームワーク概要
  /quantum             量子力学と Being
  /philosophy          哲学的アプローチ
  /divination          占いと自己理解
  /energy              エネルギーを読む
/service               サービス概要
  /session             個別相談
  /self-coaching       セルフコーチング土台作り
  /corporate           企業向け業務委託
```

### 4-2. グローバル要素

- **ヘッダー：** ロゴ（縁 enishi）+ ナビ + LINE ボタン×2（個人 / 経営者）
- **フッター：** 2つの LINE 窓口リンク
- **全ページ共通：** ページ末尾に LINE 誘導 CTA（個人・経営者の2択）

## 6. デザイン方針

### トーン
- 知的・静謐・深み。ハウツー感・自己啓発感を出さない
- 「答えを渡す」ではなく「問いの空間を作る」デザイン

### ビジュアル方向
- ダーク or ディープなカラーパレット（深海・宇宙・森のような落ち着き）
- タイポグラフィ重視。余白を大きく取る
- アニメーションは最小限。動きより「静けさ」を大切にする

### 避けること
- 自己啓発・コーチング業界の典型的なデザイン（明るい緑・ポジティブ系）
- CTA の多用・圧力的な訴求
- 実績・資格・メディア掲載の羅列

## 8. コンテンツ方針

### 書き方の原則
- 「教える」ではなく「問いを共有する」文体
- 難解な概念も「自分ごと」として読める言葉で書く
- 1ページ = 1つの問い・1つの視点。詰め込まない

### 更新フロー
```
SNS・note で発信した内容
  ↓ 反応が良かったもの・深めたいもの
このサイトのページとして再編集・掲載
```
